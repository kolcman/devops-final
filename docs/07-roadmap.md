# План развития инфраструктуры (6 месяцев)

## Ограничения

- Небольшая команда и ограниченный бюджет.
- 60% времени команды уходит на операционные задачи.
- Выделение времени на roadmap: примерно 2 недели в месяц.

## Action items

| Что делаем | Короткое описание | Цель | Срок | Приоритет |
|---|---|---|---|---|
| IaC для VM | Перенос создания VM и сетей в Terraform | Ускорить пересоздание и снизить ручные ошибки | 3-4 недели | Высокий |
| Централизация секретов | Внедрить Vault или cloud secret manager | Убрать секреты из ручного хранения | 2 недели | Высокий |
| GitOps для конфигов | Хранить конфиги инфраструктуры в git + review flow | Прозрачность изменений и откаты | 2 недели | Высокий |
| Мониторинг VPN SLA | Добавить SLI/SLO по доступности VPN | Управление качеством сервиса | 1 неделя | Средний |
| Улучшение backup-политики | Добавить offsite-репликацию и регулярные restore drill | Снизить риск потери данных | 2 недели | Высокий |
| Hardening серверов | CIS-бейзлайн, fail2ban, ограничение SSH | Повышение безопасности | 2 недели | Средний |
| Документация и onboarding | Шаблоны runbook и checklist для новых инженеров | Снижение bus factor | 1 неделя | Средний |

## Очередность

1. IaC для VM
2. Централизация секретов
3. Улучшение backup-политики
4. GitOps для конфигов
5. Hardening и SLO
6. Документация/onboarding
