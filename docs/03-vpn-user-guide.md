# Руководство пользователя VPN

Это руководство для сотрудников, которым нужен доступ во внутреннюю сеть компании через OpenVPN.

## 1. Зачем нужен VPN

VPN нужен для безопасного подключения к внутренним сервисам компании из интернета:

- шифрует трафик в публичных сетях;
- ограничивает доступ к внутренним ресурсам только сотрудниками;
- снижает риск утечки данных при удаленной работе.

## 2. Что получить от администратора

Минимальный комплект:

- файл `client.ovpn`;
- адрес VPN-сервера и порт (текущее значение: `111.88.153.158:1194/udp`);
- срок действия сертификата и дата планового продления;
- контакт для поддержки: `infra-support@company.local`.

Альтернативный формат (если не используют единый `.ovpn`):

- `ca.crt`;
- `client.crt`;
- `client.key`;
- шаблон client-конфига.

## 3. Как получить сертификат

1. Сгенерировать ключ и запрос на сертификат (CSR) на своей машине.
2. Передать администратору только CSR-файл (приватный ключ не передавать).
3. Получить от администратора:
   - подписанный клиентский сертификат;
   - CA сертификат;
   - готовый `client.ovpn` или шаблон с параметрами.
4. Сохранить полученные файлы в каталог `~/vpn-client`.

## 4. Подключение на Linux (Ubuntu/Debian)

1) Установить клиент:

```bash
sudo apt-get update
sudo apt-get install -y openvpn
```

2) Подключиться:

```bash
cd ~/vpn-client
sudo openvpn --config client.ovpn
```

3) Успешное подключение:

- в выводе есть `Initialization Sequence Completed`.

## 5. Подключение на Windows/macOS (кратко)

- Установить официальный OpenVPN клиент (OpenVPN Connect).
- Импортировать `client.ovpn`.
- Нажать Connect.
- Проверить, что статус подключения Active.

## 6. Проверка, что VPN работает

1) До подключения:

```bash
curl ifconfig.me
```

2) После подключения (в другом терминале):

```bash
curl ifconfig.me
```

Ожидаемо: внешний IP меняется на IP VPN-сервера.

## 7. Как отключиться

- Linux CLI: в терминале с OpenVPN нажать `Ctrl+C`.
- OpenVPN Connect: нажать Disconnect.

## 8. Типовые проблемы

### Ошибка сертификата

- Проверьте срок действия сертификата.
- Проверьте, что используется актуальный `client.ovpn`.
- При необходимости запросите перевыпуск у администратора.

### Нет доступа к внутренним сервисам после подключения

- Проверьте, что туннель поднят (`Initialization Sequence Completed`).
- Проверьте DNS/маршруты (переподключитесь).
- Обратитесь в поддержку с логом клиента.

### Частые разрывы соединения

- Проверьте стабильность интернета.
- Попробуйте другую сеть.
- Передайте администратору время и логи разрыва.

## 9. Куда писать при проблемах

- Email: `infra-support@company.local`
- Тема: `VPN issue: <кратко>`
- В письме указать:
  - ОС и версию клиента;
  - время возникновения проблемы (UTC);
  - последние 30-50 строк лога OpenVPN.
