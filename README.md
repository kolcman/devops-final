# Финальный проект: Старт в DevOps — системное администрирование

Мини‑инфраструктура в облаке для небольшой компании: **удостоверяющий центр (PKI)**, **VPN (OpenVPN)**, **мониторинг (Prometheus/Alertmanager)**, **резервное копирование**, плюс **документация** и **план развития**. Основной фокус — **воспроизводимость** (скрипты, deb‑пакеты, идемпотентность) и готовность к эксплуатации.

## Что реализуется

- **PKI (Easy‑RSA)**: развёртывание CA, выпуск корневого сертификата, автоматизация и упаковка конфигов.
- **VPN (OpenVPN)**: серверные/клиентские сертификаты, генерация клиентских конфигов, проверка подключения.
- **Мониторинг**: Prometheus + экспортёры + алерты на PromQL + Alertmanager (email‑уведомления).
- **Бэкап**: надёжное хранение данных и артефактов (скрипты/deb‑пакеты/конфиги) + мониторинг бэкап‑компонента.
- **Документация**: схемы, user guide для VPN, admin guide, описание мониторинга/алертов и бэкапа.
- **Roadmap**: план улучшений инфраструктуры на ~6 месяцев.

## Быстрый “smoke test” (что должно быть проверено)

- **Воспроизводимость**: скрипты/пакеты работают на *новой* VM (или после пересоздания) без ручных “доправок”.
- **VPN**: клиент подключается; внешний IP меняется на IP VPN‑сервера.
- **Мониторинг**: метрики приходят; алерты определены; уведомления уходят через Alertmanager.
- **Безопасность**: открыты только нужные порты; доступ к экспортёрам ограничен (например, только от Prometheus‑сервера).
- **Бэкап**: данные и артефакты хранятся надёжно; есть план действий при отказах; бэкап‑компонент мониторится.
